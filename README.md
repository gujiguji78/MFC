# MyDrawBoard —— 简易绘图板（Project 2）

## 项目简介

MyDrawBoard 是一个基于 **MFC（Microsoft Foundation Classes）** 与 **GDI** 技术实现的简易矢量绘图程序。  
本项目作为《高级程序设计》课程项目二，旨在综合运用 **C++ 面向对象编程、设计模式与软件工程方法**，完成一个结构清晰、功能完整、具有一定工程质量保障的桌面应用程序。

---

## 主要功能（Functional Requirements）

### 1. 基本绘图功能
- 支持使用鼠标绘制 **直线、矩形、椭圆** 三种基本图形
- 绘制过程支持实时预览

### 2. 图形属性设置
- 支持自定义绘图颜色
- 支持多种线宽设置（1 / 3 / 5 / 8 像素）

### 3. 撤销与重做
- 支持 **Ctrl + Z（撤销）**
- 支持 **Ctrl + Y（重做）**
- 基于命令模式实现，可多步撤销与重做

### 4. 文件保存与读取
- 支持将绘图内容保存到本地文件
- 支持重新打开文件并完整恢复绘图内容（图形类型、颜色、线宽等）

### 5. 多语言界面
- 支持 **中文 / English / 日本語** 三种界面语言
- 支持运行时动态切换，无需重启程序

---

## 非功能特性（Non-Functional Requirements）

### 1. 软件架构
- 采用 **MFC Document/View 架构**
- 数据与界面分离，提升可维护性

### 2. 设计模式
- **工厂模式（Factory Pattern）**：用于创建不同类型的图形对象
- **命令模式（Command Pattern）**：用于实现撤销与重做功能
- **策略模式（Strategy Pattern）**：用于不同图形的绘制策略

### 3. 性能与体验
- 使用 **双缓冲绘图（Double Buffering）** 防止界面闪烁
- 状态栏实时显示鼠标坐标，提升用户交互体验

### 4. 稳定性与异常处理
- 文件读写过程加入异常捕获机制
- 防止错误文件导致程序崩溃

### 5. 单元测试
- 使用 **Google Test** 对核心逻辑进行单元测试
- 覆盖工厂模式、命令模式及序列化逻辑
- 覆盖率满足课程要求（≥ 70%）
